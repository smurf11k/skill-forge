# Довідка Markdown

Контент уроків у SkillForge пишеться в Markdown і рендериться з розширеннями GitHub-flavored Markdown.

## Підтримувані можливості

- заголовки та текст абзаців
- виділення, inline code і посилання
- впорядковані та невпорядковані списки
- таблиці
- fenced code blocks
- task lists
- alert callouts

## Заголовки і текст

### Соурс-код

```md
# Назва уроку

## Назва секції

### Підсекція
```

## Списки і задачі

### Соурс-код

```md
- Зрозуміти концепцію
- Виконати приклад

1. Відкрити курс
2. Прочитати урок
3. Надіслати квіз

- [x] Вступ завершено
- [ ] Фінальна вправа
```

## Блоки коду

Використовуйте fenced code blocks і вказуйте мову для syntax highlighting.

### Соурс-код

````md
```js
function scoreToPercent(score, total) {
  if (!total) return 0;
  return Math.round((score / total) * 100);
}
```
````

### Рендер

```js
function scoreToPercent(score, total) {
  if (!total) return 0;
  return Math.round((score / total) * 100);
}
```

## Виноски

Підтримуються виноски (callouts) у стилі GitHub.

### Соурс-код

```md
> [!NOTE]
> Використовуйте цей урок як базу перед квізом.

> [!TIP]
> Тримайте кожну секцію зосередженою на одній концепції.

> [!WARNING]
> Зміна порядку опублікованого контенту впливає на навчальний потік.

> [!CAUTION]
> Видалення квізу також видаляє його питання.
```

### Рендер

> [!NOTE]
> Використовуйте цей урок як базу перед квізом.

> [!TIP]
> Тримайте кожну секцію зосередженою на одній концепції.

> [!WARNING]
> Зміна порядку опублікованого контенту впливає на навчальний потік.

> [!CAUTION]
> Видалення квізу також видаляє його питання.

## Рекомендації для авторингу

- Робіть секції уроків короткими та зручними для швидкого перегляду.
- Віддавайте перевагу одній ідеї на один заголовок.
- Розміщуйте практичні приклади поруч із теорією.
- Використовуйте callouts для політик, ризиків або коротких порад.
