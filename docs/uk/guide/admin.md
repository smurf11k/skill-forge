# Посібник для адміністратора

Цей посібник охоплює всі адміністраторські робочі процеси в SkillForge.

## Карта адмін-маршрутів

Основні сторінки:

- `/admin` dashboard
- `/admin/team` прогрес команди
- `/admin/users` користувачі та запрошення
- `/admin/content` керування контентом
- `/admin/courses/:id` редактор окремого курсу
- `/admin/assignments` керування призначеннями

Доступ до цих маршрутів мають лише користувачі з роллю `admin`.

## Адмінський dashboard

Dashboard підсумовує стан платформи метриками на кшталт:

- індикаторів завершення
- сигналів про навчання в зоні ризику
- знімків продуктивності на рівні курсу

## Прогрес команди

Прогрес команди агрегує навчання співробітників по курсах:

- глибину завершення уроків
- результати проходження квізів
- контекст середнього балу

Використовуйте цю сторінку для коучингу й подальшого планування.

## Користувачі та запрошення

Адміністратори можуть:

- переглядати користувачів
- змінювати ролі користувачів
- видаляти користувачів
- створювати посилання або email-запрошення
- повторно надсилати запрошення
- відкликати запрошення в очікуванні

Прийняття запрошення відбувається публічно через `/accept-invite`.

## Керування призначеннями

Сторінка призначень підтримує:

- призначення курсів одному або багатьом користувачам
- встановлення і редагування дедлайнів
- моніторинг статусів призначень
- видалення застарілих призначень

Призначення формують сигнали щодо дедлайнів, які показуються на картках курсів у співробітника.

## Керування контентом

Робоча область контенту дає CRUD для:

- курсів
- уроків
- квізів
- питань квізів

Кожен тип контенту підтримує керування станом публікації.

## Редактор курсу

Для кожного курсу є окремий редактор, де адміністратор може:

- переглядати уроки й квізи в єдиному впорядкованому списку
- змінювати порядок перетягуванням
- швидко переходити до вбудованих сценаріїв редагування

## Авторинг уроків

Контент уроків пишеться в Markdown і підтримує:

- багаті текстові структури
- таблиці та code blocks
- alert callouts

Адміністратор може писати й попередньо переглядати Markdown перед публікацією.

## Модель квізів і питань

Квізи містять впорядковані питання.

Поточна модель питань використовує одну правильну відповідь на питання.

## Життєвий цикл чернетки та публікації

Рекомендований процес релізу:

1. Створити курс як чернетку
2. Додати уроки й квізи
3. Підготувати Markdown і питання квізів
4. Відсортувати послідовність
5. Перевірити в адмінському процесі попереднього перегляду
6. Опублікувати контент для співробітників
